00

How to use this

Each milestone is a self-contained prompt. The loop is the same every time:

  1. Copy the milestone prompt into your coding agent (Claude Code / Cursor). Each one names the exact libraries, files, and acceptance criteria, and explicitly says what not to build yet.
  2. Review the diff / PR. Prompts are scoped to land in one reviewable chunk and reference the shared types in packages/shared so the mobile app and API never drift.
  3. Run the acceptance checks listed on the card before moving on. Each milestone leaves main in a working state.
  4. Move to the next milestone. Dependencies are noted on every card — don't skip ahead past a needs: line.
01

Recommended stack

TypeScript end-to-end so types and validation schemas are shared between the app and the API. Everything here is chosen to reach a working MVP fast while leaving room for the rewards + partner-portal work in Phase 3.

02

Architecture

A standard mobile-client + REST-API + Postgres core, with two manamap-specific wrinkles: a Redis-backed presence layer for "who's here right now", and connection-gated contact data.

03

Data model

PostgreSQL with the PostGIS extension for store geo-queries. Modeled in Prisma; geom columns use raw SQL / a PostGIS helper.

04

API surface

REST under /v1. Auth via short-lived JWT access token + rotating refresh token stored in expo-secure-store. Request/response bodies validated with shared zod schemas.

05

Build sequence

Foundations first, then the flow you prototyped, then community and rewards. Each card's prompt is paste-ready.
