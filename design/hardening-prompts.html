Build prompts & a checklist.

Grounded build prompts for manamap, written against the actual modules and file paths. M14–M18 are shipped (verified in the codebase); M19 is the next one up — the moderation dashboard that closes the M14 safety loop. Apple auth was already built, so it's a verification checklist, not a prompt.

Already done — Apple Sign In

Sign in with Apple is fully implementedNo prompt needed

The API exposes POST /v1/auth/apple (apple.service.ts verifies the identity token against Apple's JWKS; auth.service.ts upserts the user). The mobile SignInScreen.tsx already renders the native Apple button on iOS and calls it. AppleAuthBodySchema exists in packages/shared. This means you're not exposed to the App Store "must offer Sign in with Apple" rejection I flagged earlier — good. Just verify the config:

Set APPLE_CLIENT_ID in apps/api/.env and ensure it's in config/config.schema.ts — the service throws ServiceUnavailable if it's missing.
Enable the iOS capability — ios.usesAppleSignIn: true in app.config.ts, and confirm the expo-apple-authentication plugin is registered.
Apple Developer portal — turn on "Sign In with Apple" for the app's bundle ID; the APPLE_CLIENT_ID (audience) must match.
Email-on-first-login — Apple returns email only once. auth.service.ts already handles "email required on first sign-in," and the client requests the EMAIL scope. Nothing to change; just don't strip it.
Android — the Apple button is correctly gated to Platform.OS === 'ios'; Discord stays the Android path.
